# GOAL — רמה B: מוכן להפצה ולשימוש של משתמשים רבים

> ממשיך מ-[GOAL.md](GOAL.md) (רמה A — "עובד אצלי", **הושג ✅**). מסמך זה מגדיר מתי התוסף מוכן ש**חברים רבים** יתקינו וישתמשו בו באמינות, בלי ידע טכני. נמדוד מולו.

---

## 🎯 תמונת הסיום (רמה B)

חבר/ה מהאוניברסיטה מקבל/ת קישור, מתקין/ה את התוסף **בלחיצה אחת** מחנות Chrome, פותח/ת הרצאת Moodle, לוחץ/ת על הכפתור — וכתוביות עברית **איכותיות** מופיעות. **בלי להתקין שרת, בלי להגדיר endpoint, בלי לדעת מה זה Whisper.** עובד גם כשכמה משתמשים מחוברים במקביל.

---

## ✅ בדיקת הקבלה — רמה B

1. **משתמש אחר** (לא אתה), על מחשב אחר (Windows/Mac), מתקין את התוסף **בלי Developer mode** — מהחנות או מחבילה מוכנה.
2. ברירת המחדל כבר מצביעה על **השרת המשותף** (`wss://...`) — המשתמש לא נוגע בהגדרות.
3. פותח **הרצאת Moodle אמיתית** (לא רק YouTube) → לוחץ → כתוביות עברית איכותיות חיות.
4. **3-5 משתמשים במקביל** מקבלים כתוביות בלי קריסה או האטה חמורה.
5. שרת/רשת נופלים → המשתמש רואה **הודעת שגיאה ברורה** (לא כישלון שקט), ויש התאוששות אוטומטית כשחוזר.
6. הסרטון נגמר או מושהה → הלכידה **נעצרת אוטומטית**.
7. קיימת **מדיניות פרטיות גלויה** שמסבירה שאודיו ההרצאה נשלח לשרת.

**אם 1-7 עוברים → רמה B הושגה.**

## 🧩 מה צריך להשלים (מעבר לרמה A)

### 1. שרת תמלול משותף — הלב של רמה B
| דרישה | פירוט |
|-------|--------|
| חומרה | GPU עם CUDA + מודל עברי מלא (`ivrit-ai/whisper-large-v3-turbo-ct2`) |
| אבטחה | **`wss://` עם TLS** (nginx/caddy reverse-proxy) — חובה, כי Moodle הוא HTTPS ([CONTRACTS.md](CONTRACTS.md)) |
| מקביליות | מספר sessions בו-זמנית (כל משתמש = WebSocket) + בדיקת עומס |
| זמינות | אירוח always-on: ענן GPU (RunPod/Vast.ai) או מכונה ביתית עם uptime |
| עלות | מי משלם, כמה לשעה, האם לכבות בשעות מתות |

### 2. הפצת התוסף
- **endpoint ברירת מחדל = השרת המשותף** (כדי שאף אחד לא יצטרך להגדיר ידנית).
- **icons אמיתיים** (להחליף את ה-placeholder הנוכחי).
- **פרסום ל-Chrome Web Store**: תיאור, צילומי מסך, מדיניות פרטיות, דמי רישום מפתח חד-פעמיים (~$5). נותן התקנה בלחיצה + עדכונים אוטומטיים.
- **או** הפצה כחבילה (`.zip`/unpacked) עם הוראות — פחות נוח, בלי auto-update.

### 3. עמידות ו-UX למשתמש לא-טכני
- טיפול בשגיאות: שרת לא זמין → הודעה ברורה במקום כלום.
- **reconnection** אוטומטי אם ה-WebSocket נופל.
- בדיקה על **פריסות Moodle אמיתיות** (iframe, נגנים שונים) — לא רק YouTube.
- עצירה אוטומטית בסוף/השהיית סרטון.
- אינדיקציה ברורה שהלכידה פעילה.

### 4. פרטיות ומשפטי
- **מדיניות פרטיות** (חובה ב-Chrome Web Store): מה נשלח, מה נשמר/לא נשמר, לאיזה שרת.
- גילוי למשתמש לפני לכידה ראשונה.

## 📊 איפה אנחנו ביחס לרמה B

### ✅ הושלם (קוד/תשתית מוכנים)
- **עמידות (קריטריונים 5-6):** reconnection אוטומטי עם backoff, הודעות שגיאה ברורות למשתמש (באנר), עצירה אוטומטית כשהסרטון מושהה/נגמר.
- **ריבוי משתמשים (קריטריון 4):** השרת מטפל בחיבורים מקבילים (נעילת מודל); נבדק עם 2 לקוחות במקביל.
- **שרת `wss://` turnkey (קריטריונים 1-3, תשתית):** `Dockerfile` + `docker-compose.yml` + `Caddyfile` (TLS אוטומטי) + [`backend/DEPLOY.md`](backend/DEPLOY.md).
- **פרטיות (קריטריון 7):** [`PRIVACY.md`](PRIVACY.md).
- **אריזה והפצה:** `tools/package-extension.ps1` (zip לחנות) + [`STORE_LISTING.md`](STORE_LISTING.md).
- **endpoint:** `server.py` תומך ב-GPU+מודל עברי דרך env; ברירת המחדל בתוסף מסומנת לשינוי לפני אריזה.

### ⏳ נותר — פעולות שרק אתה יכול לעשות
1. **להקים את השרת המשותף** — שכור מכונה (GPU מומלץ), הצבע דומיין, `docker compose up` (לפי DEPLOY.md). _(עולה כסף)_
2. **לקבוע את ה-endpoint** ל-`wss://<server>` ב-`service-worker.js` ולארוז.
3. **צילומי מסך + פרסום ל-Chrome Web Store** (חשבון מפתח ~$5, אישור גוגל).
4. **לבדוק עם 3-5 חברים אמיתיים** (בדיקת הקבלה 1-7 על מחשבים אחרים).
