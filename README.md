# 🎬 Subtitle Sidekick — כתוביות עברית להרצאות

  <img src="https://img.shields.io/badge/Python-3.12-blue" alt="Python 3.12" />

*A local, free, privacy-first desktop app that transcribes recorded Hebrew lectures into perfectly-synced subtitles — no cloud, no account, no cost.*

אפליקציה שמתמללת הרצאות (Moodle / כל וידאו) לעברית ומציגה כתוביות **מסונכרנות בול לאודיו**. רץ מקומית על המחשב שלך — בלי ענן, בלי חשבונות, בלי עלות.

> מבוסס Whisper עם מודל עברי ייעודי (ivrit-ai). אומת: מתמלל עברית במדויק, כולל פיסוק.

## ⭐ תכונות עיקריות

- 🏠 **מסך בית** — סטטיסטיקה (הרצאות, קורסים, נצפו/לא נצפו) וטיפים ללמידה.
- 📚 **ספריית קורסים** — כל הרצאה מאורגנת לפי קורס, עם תפריט פעולות לכל הרצאה.
- ⬇️ **הורדה מקישור** (YouTube / Moodle) — נכנס אוטומטית לתור התמלול.
- 🗂️ **תור תמלול** — אפשר לגרור כמה קבצים יחד, מתמללים אחד-אחד ברקע.
- ⚡ **זיהוי GPU אוטומטי** — NVIDIA מזורז אוטומטית; בלי GPU, מתג "מהיר" למודל קליל יותר.
- 🔎 **חיפוש בכתוביות** + קפיצה ישירה לרגע בהרצאה.
- ✏️ **עורך כתוביות** מובנה לתיקון טעויות תמלול.
- 📄 **ייצוא** לתמליל TXT / Word.
- ⛶ **נגן מלא**: מסך מלא, דילוג 10 שניות, מהירות ניגון 0.5x–2x.
- 🌐 **נגן עצמאי בדפדפן** — נוצר לכל הרצאה, עובד בלי האפליקציה.
- 🔒 **100% מקומי** — בלי ענן, בלי חשבון, האודיו לא עוזב את המחשב.

## 📥 התקנה (פעם אחת)

1. **התקן Python 3.12** — [הורדה מכאן](https://www.python.org/downloads/release/python-3120/). בהתקנה **סמן ✅ "Add Python to PATH"**.
2. **הורד את הפרויקט** — כפתור ירוק **Code** → **Download ZIP** → חלץ לתיקייה.
3. **לחיצה כפולה על `run.bat`** — זה מתקין הכל לבד (כמה דקות, פעם אחת) ויוצר אייקון **"Subtitle Sidekick"** בשולחן העבודה.

זהו. מהפעם הבאה — פשוט לחיצה כפולה על האייקון בשולחן העבודה. **בלי שורת פקודה.**

> בהרצה הראשונה יורד המודל העברי (~1.5GB) אוטומטית. אם יש כרטיס מסך NVIDIA — האפליקציה תזהה ותשתמש בו אוטומטית (מהיר בהרבה).

## ⭐ איך משתמשים

1. **הורד הרצאה** — או הדביקו קישור ישיר (YouTube / Moodle / כל קישור-וידאו ישיר) ולחצו **הורד**, או הורידו בעצמכם (למשל בעזרת תוסף הכרום Video DownloadHelper).
2. פתח את האפליקציה (האייקון בשולחן העבודה, או סימנייה ל-`subsidekick://open` בכרום) — תיפתח **למסך הבית**.
3. מהבית: **הרצאה חדשה** → **גררו או בחרו** קובץ (אפשר כמה יחד — נכנסים ל**תור** שמתמלל סדרתית) → **צור כתוביות** → **▶ צפה עם כתוביות**.

כל הרצאה שתומללה נרשמת אוטומטית לתפריט הצד (☰) — מאורגנת לפי קורסים, עם תפריט פעולות (⋯) לכל הרצאה: שינוי שם, העברה לקורס אחר, פתיחה בדפדפן, הסרה.

> למחשב איטי בלי GPU: מתג **"מהיר"** מתמלל פי כמה מהר (מודל קל יותר). הרצאה ארוכה על מעבד לוקחת זמן — אפשר להשאיר רץ ברקע (שבשבת ההתקדמות גלויה מכל מסך).

## 🛠️ אתגרים טכניים שנפתרו

<details>
<summary>פירוט עבור מי שמתעניין בצד ההנדסי (לחצו להרחבה)</summary>

- **באג רקורסיה אינסופית ב-pywebview:** חשיפת אובייקט החלון (`Window`) על מחלקת ה-API שמשמשת כגשר ל-JS גרמה ל-pywebview לסרוק אותו רקורסיבית בעת בניית הגשר — קריסה/תקיעה אקראית, בעיקר בהזזת החלון. אותרה ע"י קריאת מקור הספרייה, ותוקנה ע"י קונבנציית `_` בשם התכונה (ש-pywebview מדלג עליה אוטומטית).
- **חסימת ניגון וידאו מ-`file://`:** pywebview מגיש את ה-UI דרך שרת HTTP מקומי, והדפדפן חוסם טעינת מדיה מ-`file://` כשהמקור הוא `http` (מסך שחור, 0:00 קפוא). הפתרון: שרת מדיה מקומי פנימי ב-Python שתומך ב-HTTP Range (כדי לאפשר קפיצה בווידאו), עם URL ייעודי שמוזרם ל-JS.
- **מסך מלא שמסתיר את הפקדים:** ה-Fullscreen API חל רק על האלמנט שקיבל `requestFullscreen()` ועל צאצאיו — פקדי הנגן (פליי/התקדמות/מהירות) שהיו מחוץ לאלמנט נשארו מאחור. הועברו להיות *בתוך* אלמנט המסך כ-overlay שקוף.
- **מדידת ביצועים על חומרה אמיתית (CPU בלבד, בלי GPU נתמך):** השוואה בין מודל "מדויק" ל-"מהיר" על קליפ של 7.7 דקות — הראתה פער של פי 7-8+ בזמן ריצה, מה שהוביל להחלטה לתת ברירת מחדל מודעת-חומרה במקום הנחה גנרית.

</details>

```bash
py -3.12 -m venv .venv && .venv\Scripts\activate    # macOS/Linux: python3.12 -m venv .venv && source .venv/bin/activate
```

```
requirements.txt            תלויות
tools/
  transcribe_to_srt.py      תמלול משורת פקודה: קובץ → SRT
  player.html               נגן מקומי לצפייה עם כתוביות
  make_shortcut.py          יוצר את האייקון בשולחן העבודה
  register_protocol.py      רושם subsidekick:// כדי שסימנייה בכרום תפתח את האפליקציה
runpod_server/               קוד השרת (Docker image) למצב "שרת מהיר" האופציונלי
  handler.py                פונקציית התמלול שרצה על השרת
  Dockerfile                בניית ה-image (אופה את המודל העברי בתוכו)
```

## ☁ מצב "שרת מהיר" (אופציונלי)

<details>
<summary>תמלול דרך שרת GPU שלכם — מהיר יותר, אבל לא חובה (לחצו להרחבה)</summary>

ברירת המחדל היא ותישאר **מקומית** — בלי קונפיג, בלי חשבון. מי שרוצה תמלול מהיר יותר יכול לבחור **"☁ שרת GPU (ענן)"** במסך ההעלאה, אחרי שמקימים שרת אישי ב-RunPod (חד-פעמי, כ-15–20 דקות).

חשוב: **זה לא שרת משותף**. לכל משתמש endpoint ו-API key אישיים משלו ב-RunPod — אין שום כתובת או מפתח מוטבעים באפליקציה.

### מה צריך מראש (פעם אחת, אם אין כבר)

- ☑ חשבון [RunPod](https://www.runpod.io/) עם קרדיט (אפילו $5-10 מספיקים להתחלה).
- ☑ [Docker Desktop](https://www.docker.com/products/docker-desktop/) מותקן ופתוח.
- ☑ חשבון חינמי ב-[Docker Hub](https://hub.docker.com/) (להעלאת ה-image).

> ⚠️ **אם תיקיית הפרויקט נמצאת תחת OneDrive** (כמו `...\OneDrive\...`): Docker עלול להיכשל בקריאת קבצים שסונכרנו כ"placeholder בענן" עם שגיאה כמו `invalid file request Dockerfile`. אם זה קורה — העתיקו את תיקיית `runpod_server` למקום מחוץ ל-OneDrive (למשל `C:\runpod_server`) ובנו משם.

### שלב 1 — מפתח API ב-RunPod

1. נכנסים ל-[runpod.io](https://www.runpod.io/) ומתחברים.
2. תפריט צד → **Settings** → לשונית **API Keys** → **+ Create API Key**.
3. נותנים שם (לדוגמה `subtitle-sidekick`) → **Create**.
4. **מעתיקים את המפתח עכשיו** — הוא מוצג פעם אחת בלבד.

### שלב 2 — בניית השרת והעלאתו ל-Docker Hub

בטרמינל (PowerShell/cmd), מתוך תיקיית הפרויקט:

```bash
cd runpod_server
docker login
docker build -t YOUR_DOCKERHUB_USERNAME/subtitle-sidekick-server:latest .
docker push YOUR_DOCKERHUB_USERNAME/subtitle-sidekick-server:latest
```

בשורות האלה, מחליפים את `YOUR_DOCKERHUB_USERNAME` בשם המשתמש שלכם ב-Docker Hub (למשל אם שם המשתמש הוא `dani123`, השם המלא יהיה `dani123/subtitle-sidekick-server:latest`).

הסבר לכל שורה:

- **שורה ראשונה** (`docker login`) — מבקשת שם משתמש וסיסמה של Docker Hub; חד-פעמי.
- **שורה שנייה** (`docker build`) — מורידה את המודל העברי (כמה GB) ובונה את ה-image. **לוקח זמן, אפילו 20-30 דקות**, בהתאם למהירות האינטרנט. זה תקין, אפשר להשאיר רץ ברקע.
- **שורה שלישית** (`docker push`) — מעלה את ה-image המוכן לחשבון שלכם ב-Docker Hub.

### שלב 3 — יצירת ה-Endpoint ב-RunPod

1. נכנסים ב-RunPod לתפריט הצד → **Serverless** → **+ New Endpoint**.
2. בוחרים **Custom Source**.
3. בשדה ה-image מדביקים בדיוק את השם המלא שדחפתם בשלב 2 (`YOUR_DOCKERHUB_USERNAME/subtitle-sidekick-server:latest`).
4. בוחרים GPU — אופציה זולה שמספיקה (RTX A4000 או A5000), אין צורך ב-GPU חזק לתמלול.
5. קובעים Workers: כך ש-Min Workers = `0` (לא משלמים כשלא משתמשים) ו-Max Workers = `1` (מספיק לשימוש אישי).
6. לוחצים על כפתור **Create Endpoint**.
7. בדף ה-endpoint שנפתח, בראש העמוד מופיע **Endpoint ID** (מחרוזת אותיות/ספרות) — מעתיקים אותו.

### שלב 4 — הזנת הפרטים באפליקציה

ב-Subtitle Sidekick: ⚙ **הגדרות** → אזור "שרת ענן":
- **כתובת השרת:** `https://api.runpod.ai/v2/ENDPOINT_ID` (מחליפים `ENDPOINT_ID` במה שהעתקתם בשלב 3.7)
- **מפתח API:** המפתח משלב 1.4
- **שמירת הגדרות השרת**.

זהו — מהפעלה הבאה, בוחרים **☁ שרת GPU (ענן)** מתפריט מצב התמלול בהעלאת הרצאה. ה-API key נשמר במחשב ולא צריך להזין אותו מחדש.

**עלות:** RunPod Serverless מחייב רק על שניות עיבוד בפועל (scale-to-zero — אין חיוב כשהשרת לא בשימוש). העלות המצטברת מוצגת בהגדרות, לפי מחיר לשעה שתזינו.

אם השרת לא מוגדר, לא זמין, או מחזיר שגיאה — תוצג הודעה ברורה, ואפשר לבחור מצב מקומי ולהמשיך בלי לאבד את הקובץ.

</details>
